# Universal Ambient Light

**Универсальное приложение для фоновой подсветки экрана (Ambilight) на устройствах Android**

Universal Ambient Light — это современное Android-приложение, которое анализирует изображение на экране в реальном времени и передает данные на LED-контроллеры для создания эффекта фоновой подсветки (Ambilight). Приложение полностью совместимо с Android 8.0+ и поддерживает работу как на смартфонах/планшетах, так и на Android TV.

## ✨ Основные возможности

- 🎨 **Универсальная поддержка контроллеров**: Hyperion, WLED, Adalight.
- 📱 **Оптимизация для Android TV и мобильных устройств**: удобный интерфейс для пультов ДУ (D-pad) и сенсорных экранов.
- 📷 **Режим захвата с камеры**: альтернативный метод захвата с использованием камеры устройства и коррекцией перспективы для телевизоров без поддержки захвата экрана.
- ⚙️ **Гибкая настройка**: управление качеством захвата, частотой кадров, сглаживанием цветов и задержками.
- 🚀 **Автозапуск**: автоматический старт сервиса при включении устройства.
- 🔌 **Автоматическое восстановление соединения**: попытки переподключения при потере связи с контроллером.

## 🎯 Поддерживаемые контроллеры

### Hyperion
- Полная поддержка протокола Hyperion.
- Настройка приоритета сообщений.
- Совместимость со всеми версиями Hyperion NG.

### WLED
- Поддержка протоколов **DDP** (рекомендуется для WLED 0.11+) и **UDP Raw**.
- Настройка порядка цветов (RGB, GRB, BRG и др.).
- Поддержка RGBW лент.
- Управление яркостью.

### Adalight
- Поддержка протоколов **ADA**, **LBAPA** (LightBerry APA102), **AWA** (Hyperserial).
- Настройка скорости передачи (Baud Rate).
- Подключение через USB OTG.

## 📋 Системные требования

- Android 8.0 (API 26) или выше.
- Разрешение на захват экрана (MediaProjection).
- Доступ к локальной сети (для Hyperion/WLED) или поддержка USB Host (для Adalight).

## ⚙️ Настройка

### 1. Подключение
1. Запустите приложение и откройте **Настройки**.
2. Выберите **Тип подключения**: Hyperion, WLED или Adalight.
3. Укажите IP-адрес и порт (для сетевых контроллеров) или параметры USB (для Adalight).

### 2. Конфигурация LED
- **Конфигурация LED по сторонам**: Можно настроить каждую сторону отдельно:
   - Верх: 60 LED (по умолчанию)
   - Право: 34 LED (по умолчанию)
   - Низ: 60 LED (по умолчанию)
   - Лево: 34 LED (по умолчанию)
- **Раскладка LED**: Настройка начального угла, направления (по/против часовой стрелки) и включение/выключение отдельных сторон.

### 3. Настройки захвата
- **Источник захвата**: Выберите между режимом **Экран** (по умолчанию) или **Камера**.
  - *Экран*: Стандартный захват экрана через MediaProjection API (требует разрешения на захват экрана).
  - *Камера*: Альтернативный метод с использованием камеры устройства и коррекцией перспективы. Идеально для телевизоров без поддержки захвата экрана или для внешних телефонов.
- **Частота захвата (FPS)**: По умолчанию 30 FPS (варианты: 10, 15, 24, 30, 60).
- **Качество захвата**: 
  - *Низкое (64px)* — для слабых устройств или устранения задержек.
  - *Среднее (128px)* — оптимально для большинства устройств (по умолчанию, рекомендуется).
  - *Высокое (256px)* — только для мощных процессоров.
  - *Ультра (512px)* — максимальное качество для высокопроизводительных устройств.
- **Отправлять средний цвет**: Включите для максимальной производительности (передает один цвет на всю ленту).

### 3.1. Настройка режима камеры
При использовании **Камера** в качестве источника захвата:
1. Предоставьте разрешение на **Камеру** при запросе.
2. Откройте **Настройки** → **Настройка углов камеры**.
3. Расположите устройство так, чтобы камера видела экран телевизора.
4. Перетащите четыре угловых маркера (TL, TR, BR, BL), чтобы совместить их с краями экрана телевизора.
5. Нажмите **Сохранить**, чтобы сохранить позиции углов.
6. Приложение будет использовать эти углы для коррекции перспективы во время захвата.

**Советы:**
- Используйте живое превью на главном экране для калибровки положения устройства перед запуском захвата.
- Настройка углов помогает компенсировать несимметричное расположение камеры относительно телевизора.

### 4. Сглаживание (Smoothing)
- **Включить сглаживание**: Включено по умолчанию. Рекомендуется для плавности переходов.
- **Пресет**: "Сбалансированный" (по умолчанию). Варианты: Выключено, Быстрый отклик, Сбалансированный, Плавный.
- **Время перехода**: 200 мс (по умолчанию, диапазон 50-500 мс).
- **Задержка вывода**: 80 мс (по умолчанию, диапазон 0-1000 мс). Увеличьте для синхронизации с видео (например, для Jellyfin, если светодиодная лента опережает картинку).
- **Частота обновлений**: 25 Hz (по умолчанию, варианты: 20, 25, 30, 40, 50, 60 Hz).

### 5. Запуск
1. Предоставьте разрешение на **Запись/Трансляцию экрана**.
2. Включите захват.

### Детальная настройка контроллеров

#### Hyperion
- **Host/Port**: IP сервера и порт (стандартный `19400`).
- **Приоритет**: `100` (стандартный).
- [Документация Hyperion](https://docs.hyperion-project.org/)

#### WLED
- **Host**: IP контроллера.
- **Port**: `4048` (DDP) или `19446` (UDP Raw).
- **Протокол**: DDP (предпочтительно).
- **Цветовая схема**: Убедитесь, что порядок цветов (например, GRB) совпадает с настройками в веб-интерфейсе WLED.
- [Документация WLED](https://kno.wled.ge/)

#### Adalight (USB)
- **Baud Rate**: `115200` (стандарт) или скорость вашей прошивки.
- **Протокол**: ADA (стандарт Arduino), LBAPA (для APA102), AWA.
- [Репозиторий Adalight](https://github.com/adafruit/Adalight)

##### Arduino скетч для Adalight

Готовый скетч для Arduino, совместимый с приложением, находится в файле [`adalight-sketch.ino`](adalight-sketch/adalight-sketch.ino).

**Быстрый старт:**

1. **Установите библиотеку FastLED:**
   - В Arduino IDE: `Инструменты` → `Управление библиотеками` → найдите "FastLED" → установите

2. **Настройте скетч:**
   - Откройте файл `adalight-sketch/adalight-sketch.ino`
   - Измените константы в начале файла:
     - `DATA_PIN` — пин для подключения LED ленты (по умолчанию 6)
     - `NUM_LEDS` — количество LED в ленте (должно совпадать с настройками в приложении!)
     - `LED_TYPE` — тип LED ленты (WS2812B, WS2811, SK6812 и др.)
     - `COLOR_ORDER` — порядок цветов (GRB для WS2812B, RGB для других)
     - `BRIGHTNESS` — яркость (0-255)

3. **Подключение:**
   - LED лента DATA → пин Arduino (по умолчанию 6)
   - LED лента VCC → 5V Arduino (или внешний источник питания для длинных лент)
   - LED лента GND → GND Arduino
   - ⚠️ **Важно:** Для длинных лент (>10 светодиодов) используйте внешний источник питания 5V!

4. **Загрузка скетча:**
   - Подключите Arduino к компьютеру через USB
   - Выберите плату и порт в Arduino IDE
   - Загрузите скетч

5. **Подключение к Android:**
   - Отключите Arduino от компьютера
   - Подключите Arduino к Android устройству через USB OTG кабель
   - В приложении выберите тип подключения: **Adalight**
   - Установите Baud Rate: **115200**
   - Выберите протокол: **ADA**
   - Убедитесь, что количество LED в приложении совпадает с `NUM_LEDS` в скетче

**Пример подключения WS2812B:**
```
WS2812B DATA → Pin 6 Arduino
WS2812B VCC  → 5V Arduino (или внешний 5V)
WS2812B GND  → GND Arduino
```

**Для других типов LED:**
- **APA102 (SPI)**: Используйте библиотеку FastLED с настройкой `APA102` и протокол **LBAPA** в приложении
- **WS2811**: Аналогично WS2812B, обычно `COLOR_ORDER = RGB`
- **SK6812**: Аналогично WS2812B, обычно `COLOR_ORDER = GRB`

**Устранение проблем:**
- Если LED не загораются: проверьте подключение, убедитесь что `NUM_LEDS` совпадает в скетче и приложении
- Если цвета неправильные: измените `COLOR_ORDER` (попробуйте RGB, GRB, BRG)
- Если данные не приходят: проверьте Baud Rate (должен быть 115200), убедитесь что USB OTG кабель поддерживает передачу данных

## 📱 Особенности для Android TV
Приложение оптимизировано для управления с пульта (D-pad) и отображается в лаунчере Android TV (Leanback Launcher). Для ввода IP-адресов и настроек также удобно использовать приложение "Google TV" или "Android TV Remote" на смартфоне.

## ⚠️ Важная информация

### Для владельцев телевизоров TCL
На устройствах TCL агрессивный менеджер задач может "убивать" фоновый сервис.
**Решение:**
1. Откройте **Настройки > Приложения > Специальный доступ > Автозапуск**.
2. Разрешите автозапуск для Universal Ambient Light.
3. Либо через **"Центр безопасности"** (Safety Guard) добавьте приложение в исключения.

### Воспроизведение 4K/HDR контента
Проблема с воспроизведением видео высокого качества (2K/4K/HDR) при включенной подсветке является аппаратным ограничением многих телевизоров. Встроенные процессоры часто не справляются с одновременным декодированием тяжелого видео и захватом экрана. Это проблема, которую редко удается решить программно.

**Если вы заметили лаги или подергивание видео:**
1. Снизьте качество воспроизводимого видео до 1080p или 720p (в зависимости от возможностей ТВ).
2. Или полностью отключите подсветку в приложении на время просмотра тяжелого контента.
3. Вы можете попробовать поиграться с настройками качества захвата и FPS, но вероятность устранения проблемы невелика.

### DRM-защищенный контент
**Приложения с DRM-защитой** (такие как Netflix, Disney+, Amazon Prime Video, Кинопоиск и аналогичные стриминговые сервисы) **не будут работать** с режимом захвата экрана из-за ограничений безопасности Android. Это фундаментальное ограничение API MediaProjection в Android, которое нельзя обойти.

**Почему это происходит:**
- Android блокирует захват экрана DRM-защищенного контента для предотвращения пиратства.
- Это ограничение применяется на уровне системы и не может быть отменено приложениями.

**Решения:**
1. **Используйте режим камеры**: Переключитесь на источник захвата **Камера** в настройках. Этот метод использует камеру устройства вместо захвата экрана, поэтому работает с DRM-защищенным контентом. Вам нужно будет расположить телефон/планшет камерой к экрану телевизора.
2. **Отключите подсветку**: Выключите функцию подсветки во время просмотра DRM-защищенного контента.
3. **Используйте источники без DRM**: Смотрите контент из источников, которые не используют DRM-защиту (локальные файлы, YouTube и т.д.).

**Примечание:** Режим камеры требует правильной калибровки позиций углов для точного захвата цветов.
